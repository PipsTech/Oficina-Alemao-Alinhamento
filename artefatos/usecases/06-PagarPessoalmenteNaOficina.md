# Caso de uso 06 - PagarPessoalmenteNaOficina

## 1. Descrição
O usuário cliente, como generalização do caso de uso de efetuar a compra, vai pessoalmente à oficina efetuar a compra.
## 2. Atores
Usuário Cliente
## 3. Precondições


	1. O cliente já tem decidido o que pagará ao efetuar a compra.
 
## 4. Fluxo principal

    1. O caso de Uso se inicia quando o cliente vai à oficina.
    2. O cliente define o método de pagamento com o proprietário e efetua a compra.
    3. Caso de Uso encerrado.
	

## 5. Fluxos alternativos

### 5.1 Fluxo alternativo - Trazer, em dinheiro, um valor menor que o total por engano

| **Precondições**  | 1. O cliente já tem decidido o que pagará ao efetuar a compra. |
| --- | --- |
| Passo 1  | O fluxo Alternativo se inicia quando o cliente separa o dinheiro para levar à oficina. |
| Passo 2  | O cliente vai à oficina e no momento de efetuar a compra percebe que não tem o valor total da compra. |
| Passo 3  | O cliente volta outro dia para pagar o valor correto ou paga apenas o(s) produto(s) que encaixam no valor que trouxe à oficina. |
| Passo 4  | Fluxo Alternativo encerrado. |

### 5.2 Fluxo alternativo - O cartão de crédito ou débito apresenta erro na hora do pagamento

| **Precondições**  | 1. O cliente já tem decidido o que pagará ao efetuar a compra. |
| --- | --- |
| Passo 1  | O fluxo Alternativo se inicia quando o cliente escolhe o cartão para levar à oficina. |
| Passo 2  | O cliente vai à oficina e no momento de efetuar a compra, a máquina não aceita o pagamento e apresenta mensagem de erro. |
| Passo 3  | O cliente tenta pagar com outro cartão ou volta outro dia para efetuar a compra. |
| Passo 4  | Fluxo Alternativo encerrado. |
